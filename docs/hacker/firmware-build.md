## Critical Build Settings

Getting these wrong produces firmware the bootloader silently rejects or firmware that doesn't fit.

### 1. Preprocessor Symbol: `__SAMD21G16B__` ONLY

In Microchip Studio: `Project → Properties → Toolchain → ARM/GNU C Compiler → Symbols`

**Add:** `__SAMD21G16B__`  
**Do NOT add:** `__SAMD21J18A__`

`samd21.h` uses an `#elif` chain. If both symbols are defined, the wrong device header is selected and `nvm.c` fails to compile with:
```
error: 'NVMCTRL_RWW_EEPROM_SIZE' undeclared
```

### 2. Linker Script: `samd21g16b_flash.ld`

In the `.cproj` file (or via `Project → Properties → Toolchain → ARM/GNU C Linker → Miscellaneous`):

**Use:** `src/samd21g16b_flash.ld`  
**Do NOT use:** `src/ASF/sam0/utils/linker_scripts/samd21/gcc/samd21j18a_flash.ld`

The correct script sets `ORIGIN = 0x2000` (8 KB offset for bootloader). The wrong one sets `ORIGIN = 0x0` and overwrites the bootloader.

The `.cproj` has two `<armgcc.linker.miscellaneous.LinkerFlags>` entries inside the Release `<PropertyGroup>` — verify both point at `samd21g16b_flash.ld`.

### 3. Configuration: Release Only

Debug builds don't fit in 56 KB. Use **Release** configuration exclusively.

## Build Steps

1. Open `Firmware/Source/Defcon29.atsln` in Microchip Studio 7.0
2. In the Solution Explorer, confirm the active project is `DC29`
3. In the toolbar, select **Release** (not Debug)
4. Verify the preprocessor symbol and linker script (see above)
5. **Build → Build Solution** (F7)
6. Watch the output window — build should succeed with 0 errors
7. Output: `Firmware/Source/DC29/Release/DC29.hex`

### Expected output size

```
Program Memory Usage:   ~46,000 bytes (of 57,344 available)
Data Memory Usage:      ~3,800 bytes  (of 8,192 available)
```

If it says the program is larger than 57,344 bytes, you have Debug selected or added too much code.

## Convert .hex to .uf2

The bootloader accepts `.uf2` format. Use `uf2conv.py` from the Microsoft UF2 utilities:

```bash
python3 uf2conv.py Firmware/Source/DC29/Release/DC29.hex \
    --convert \
    --output DC29.uf2
```

The `--convert` flag is required. The `--output` flag names the output file.

Get `uf2conv.py`:
```bash
curl -O https://raw.githubusercontent.com/microsoft/uf2/master/utils/uf2conv.py
```

## Modifying the Firmware

### Safe changes (unlikely to break things)

- Timing constants in `main.c` (`DEBOUNCE_TIME`, `CHORD_SHORT_MS`, `CHORD_LONG_MS`, `EFFECT_*_STEP_MS`)
- Default colors in `reset_eeprom()` in `main.c`
- Default keymap in `main.h` (`default_keymap` array)
- LED animation logic in `update_effects()` — as long as you don't touch LED 4

### Moderate risk

- Adding new effect modes — requires updating `NUM_EFFECT_MODES` and `set_effect_mode()`
- Adding new protocol commands in `serialconsole.c` — state machine is straightforward but easy to break arg counting
- Modifying keymap parsing in `keys.c` — sentinels and offsets are subtle

### High risk / advanced

- Changing EEPROM layout — must bump `FIRMWARE_VERSION` and update all `EEP_*` offsets
- Touching USB descriptor code — usually in ASF, easy to break HID enumeration
- Badge-to-badge comms in `comms.c` — affects DEF CON game functionality

### If you change the EEPROM layout

1. Update `EEP_*` constants in `main.h`
2. Bump `FIRMWARE_VERSION` in `main.h`
3. Update `reset_eeprom()` and `reset_user_eeprom()` in `main.c`
4. The version bump triggers a full EEPROM reset on first boot — all user keymaps and LED colors reset to defaults

## Build Troubleshooting

### `error: 'NVMCTRL_RWW_EEPROM_SIZE' undeclared`

Both `__SAMD21G16B__` and `__SAMD21J18A__` are defined. Remove `__SAMD21J18A__`.

### Linker error: section `.text` will not fit in region `rom`

You're building Debug. Switch to Release.

### Warning: multiple definition of linker flags

The `.cproj` has duplicate linker entries. Open the `.cproj` in a text editor and find the Release `<PropertyGroup>` — it should have two `<LinkerFlags>` entries, both pointing to `samd21g16b_flash.ld`.

### Badge boots but does nothing (all LEDs off)

Most likely the linker script points to `ORIGIN = 0x0` — firmware started at address 0 and overwrote the bootloader's exception vector table. The only recovery is entering DFU mode and reflashing a known-good UF2.

### USB HID not enumerating after reflash

Check that the USB descriptor hasn't been accidentally modified. The ASF USB descriptor configuration is in `Firmware/Source/DC29/src/config/conf_usb.h`.
