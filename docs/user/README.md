Welcome. This section is for anyone who wants to use the badge as a productivity tool — no firmware hacking required.

## What the badge does

- **4 physical buttons** that send configurable keyboard shortcuts
- **4 RGB LEDs** that change color based on what app is in focus
- **Teams mute indicator** on Button 4: red = muted, green = live, off = not in a meeting
- **Context-aware pages**: LEDs and button actions change automatically when you switch between Outlook, Slack, Teams, and 15+ other apps

## Quick start

```bash
pip install "dc29-badge[tui,hotkey]"
dc29 flow
```

Plug in your badge, run `dc29 flow`, and your LEDs will reflect your current app context. See [Setup](setup.md) for Teams pairing and first-run details.

## Button layout

```
┌─────────┬─────────┐
│  B1     │  B2     │
│ (top-L) │ (top-R) │
├─────────┼─────────┤
│  B3     │  B4     │
│ (bot-L) │ (bot-R) │
└─────────┴─────────┘
```

The LEDs follow consistent color families across every app:

| Button | Color | Semantic |
|--------|-------|----------|
| B1 | Warm red | Destructive / delete / exit |
| B2 | Cool blue | Status / communicate / reply |
| B3 | Amber | Navigate / find / reach out |
| B4 | Green | Create / confirm / go |

**Exception:** During a Teams meeting, B4 shows your actual mute state — red when muted, green when live — overriding the normal green.
