## Button keymaps (EEPROM)

Each button stores a keymap in EEPROM — the firmware sends this as a USB HID keystroke when you press the button, regardless of whether any bridge is running.

> **Note:** When `dc29 flow` is running and an app bridge is active (Outlook, Slack, Teams meeting), button presses go to the bridge, not the EEPROM keymap. The EEPROM keymap only fires when no bridge has claimed the button.

### View current keymaps

```bash
dc29 info
```

Or use `dc29 diagnose` for a more detailed view:

```bash
dc29 diagnose
```

### Set a keymap

```bash
dc29 set-key BUTTON MODIFIER KEYCODE
```

Examples:

```bash
# Button 1 → Ctrl+Z (undo)
dc29 set-key 1 0x01 0x1D

# Button 2 → Cmd+Shift+M (Teams mute on macOS)
dc29 set-key 2 0x0A 0x10

# Button 3 → Media mute key
dc29 set-key 3 0xF0 0xE2

# Button 4 → Ctrl+Shift+M (Teams mute on Windows)
dc29 set-key 4 0x03 0x10
```

Changes are saved to EEPROM immediately and survive power cycles.

### Clear all keymaps

If buttons are firing unexpected shortcuts alongside bridge actions (double-injection), zero all keymaps:

```bash
dc29 clear-keys
```

### Modifier values

| Modifier | Value | Keys |
|----------|-------|------|
| Ctrl | `0x01` | Left Control |
| Shift | `0x02` | Left Shift |
| Alt | `0x04` | Left Alt |
| GUI/Cmd | `0x08` | Windows key / Cmd |
| Ctrl+Shift | `0x03` | |
| Shift+GUI | `0x0A` | Shift+Cmd (macOS) |
| Ctrl+Alt | `0x05` | |
| Media key | `0xF0` | (use with media keycodes below) |

### Common keycodes

| Key | Code | Key | Code |
|-----|------|-----|------|
| a–z | `0x04`–`0x1D` | Enter | `0x28` |
| 1–9 | `0x1E`–`0x26` | Escape | `0x29` |
| 0 | `0x27` | Backspace | `0x2A` |
| Space | `0x2C` | Tab | `0x2B` |
| F1–F12 | `0x3A`–`0x45` | Delete | `0x4C` |

**Letters:** keycode = `0x04 + (position in alphabet)`. So `a`=`0x04`, `m`=`0x10`, `z`=`0x1D`.

### Media keycodes (use with `mod = 0xF0`)

| Action | Code |
|--------|------|
| Mute | `0xE2` |
| Volume up | `0xE9` |
| Volume down | `0xEA` |
| Play/Pause | `0xCD` |
| Next track | `0xB5` |
| Previous track | `0xB6` |

## LED colors

### Set a single LED (temporary, RAM only)

```bash
dc29 set-led N R G B
```

Example — set LED 1 to orange:

```bash
dc29 set-led 1 255 80 0
```

This is RAM-only and resets on power cycle.

### Permanent LED colors (EEPROM)

Use the TUI's **Keys** tab or set via the serial console menu (connect with any serial terminal at 9600 baud and follow the menu).

## Config file

Create `~/.config/dc29/config.toml` to set defaults:

```toml
[badge]
port       = "/dev/tty.usbmodem14201"
brightness = 0.8

[teams]
toggle_hotkey = "<ctrl>+<alt>+m"

[teams.buttons]
1 = "leave-call"
2 = "toggle-video"
3 = "toggle-hand"
4 = "toggle-mute"

[outlook.buttons]
1 = "delete"
2 = "reply"
3 = "reply-all"
4 = "forward"
```

Generate a default config file:

```bash
dc29 config init
```

View the current effective config:

```bash
dc29 config show
```

## LED effects

The firmware has three built-in LED effect modes for LEDs 1–3 (LED 4 is always reserved for the Teams mute indicator):

| Mode | Command | Description |
|------|---------|-------------|
| Off | `dc29 set-effect 0` | Static EEPROM colors |
| Rainbow chase | `dc29 set-effect 1` | One LED at a time, cycling hues |
| Breathe | `dc29 set-effect 2` | All three LEDs fade in/out |

You can also cycle effects with the 4-button chord: hold all 4 buttons for 300 ms–2 s and release.
