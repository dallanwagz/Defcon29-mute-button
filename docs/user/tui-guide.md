## What is the TUI?

The TUI (terminal user interface) is a live companion app that shows exactly what each button does right now. Think of it as your badge's user manual that updates itself in real time.

Open it in a permanent tab in iTerm2 or your preferred terminal. It stays in sync with the badge and all active bridges.

## Launch

### With all bridges (recommended)

```bash
dc29 start
```

Runs the TUI and all bridges in one process. Bridge log messages appear in the **Log tab** instead of cluttering your terminal.

### TUI only

```bash
dc29 ui
```

Connects to the badge without starting any bridges. Use this if you're running `dc29 flow` in a separate terminal.

## Tabs

**Dashboard** — active page name, live LED colors and button labels, Teams meeting state, connection status.

**Keys** — EEPROM keymap for all 4 buttons and the slider (modifier + keycode). Read-only here; to change keymaps see [Customizing](customizing.md).

**LEDs** — live RGB swatches for all 4 LEDs, updating in real time.

**Effects** — current firmware LED effect mode (off / rainbow-chase / breathe). Click to cycle through modes.

**Log** — live stream of badge events and bridge activity. When running via `dc29 start`, all bridge output appears here.

## Quitting

Press `q` or `Ctrl+C`. When running via `dc29 start`, this stops all bridges and closes the badge connection cleanly.
