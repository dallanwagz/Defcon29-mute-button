## `dc29.protocol` — Constants Module

`dc29/protocol.py` is a pure-Python, import-safe constants module. No I/O, no side effects.

### Escape Byte

```python
ESCAPE: int = 0x01
```

### Command Constants (Host → Badge)

| Constant | Value | Description |
|----------|-------|-------------|
| `CMD_MUTED` | `0x4D` (`'M'`) | Set LED 4 red |
| `CMD_UNMUTED` | `0x55` (`'U'`) | Set LED 4 green |
| `CMD_CLEAR` | `0x58` (`'X'`) | Turn LED 4 off |
| `CMD_SET_KEY` | `0x4B` (`'K'`) | Write keymap to EEPROM |
| `CMD_QUERY_KEY` | `0x51` (`'Q'`) | Query keymap |
| `CMD_SET_LED` | `0x4C` (`'L'`) | Set LED color (RAM only) |
| `CMD_BUTTON_FLASH` | `0x46` (`'F'`) | Enable/disable button flash |
| `CMD_SET_EFFECT` | `0x45` (`'E'`) | Set LED effect mode |

### Event Constants (Badge → Host)

| Constant | Value | Description |
|----------|-------|-------------|
| `EVT_BUTTON` | `0x42` (`'B'`) | Button pressed |
| `EVT_KEY_REPLY` | `0x52` (`'R'`) | Reply to Q query |
| `EVT_KEY_ACK` | `0x41` (`'A'`) | ACK after K command |
| `EVT_EFFECT_MODE` | `0x56` (`'V'`) | Effect mode changed |
| `EVT_CHORD` | `0x43` (`'C'`) | Chord fired |

## `BadgeAPI` Class

`dc29.badge.BadgeAPI` is the thread-safe serial interface. Source: `dc29/badge.py`.

### Constructor

```python
BadgeAPI(port: str, brightness: float = 1.0)
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `port` | `str` | Serial port path, e.g. `/dev/tty.usbmodem14201` |
| `brightness` | `float` | LED brightness scale 0.0–1.0, applied to `set_led()` calls |

Opens the serial port immediately and starts the reader thread.

### Callbacks

All callbacks are invoked from the **reader thread**. Use `loop.call_soon_threadsafe()` to hand off to an asyncio event loop.

| Callback | Signature | Fires when |
|----------|-----------|------------|
| `on_button_press` | `(btn: int, mod: int, kc: int)` | Any button pressed |
| `on_key_reply` | `(btn: int, mod: int, kc: int)` | Reply to `CMD_QUERY_KEY` |
| `on_key_ack` | `(btn: int)` | Badge acknowledges `CMD_SET_KEY` |
| `on_effect_mode` | `(mode: int)` | Firmware effect mode changed |
| `on_chord` | `(type: int)` | 4-button chord (1=short, 2=long) |
| `on_connect` | `()` | Serial port opened |
| `on_disconnect` | `()` | Serial port lost |
| `on_state_change` | `(state: BadgeState)` | Any of the above (unified feed) |

Example — crossing the asyncio boundary:

```python
import asyncio
from dc29.badge import BadgeAPI

loop = asyncio.get_running_loop()
event_queue: asyncio.Queue = asyncio.Queue()

badge = BadgeAPI("/dev/tty.usbmodem14201")
badge.on_button_press = lambda btn, mod, kc: loop.call_soon_threadsafe(
    event_queue.put_nowait, ("button", btn, mod, kc)
)
```

### Properties

```python
badge.connected: bool          # True when serial port is open
badge.state: BadgeState        # current snapshot of all observable state
```

### Write Methods

All write methods are non-blocking fire-and-forget.

#### `set_mute_state(state: MuteState) -> None`

Drive LED 4 to reflect meeting mute state. Always uses full brightness (ignores the `brightness` constructor parameter).

```python
from dc29.protocol import MuteState

badge.set_mute_state(MuteState.MUTED)          # LED 4 red
badge.set_mute_state(MuteState.UNMUTED)        # LED 4 green
badge.set_mute_state(MuteState.NOT_IN_MEETING) # LED 4 off
```

#### `set_led(n: int, r: int, g: int, b: int) -> None`

Set LED `n` (1–4) to color `(r, g, b)`. Brightness scaling from the constructor is applied. **RAM only — reverts on power cycle.**

```python
badge.set_led(1, 255, 0, 0)    # LED 1 red
badge.set_led(4, 0, 200, 0)    # LED 4 green
badge.set_led(2, 0, 0, 0)      # LED 2 off
```

#### `set_key(button: int, modifier: int, keycode: int) -> None`

Write a single-key macro for `button` (1–6) to EEPROM. The badge responds with `EVT_KEY_ACK`.

```python
badge.set_key(1, 0x0A, 0x10)  # button 1: Shift+Cmd+M (macOS Teams mute)
badge.set_key(2, 0xF0, 0xE2)  # button 2: media mute
badge.set_key(1, 0, 0)        # button 1: no-op (clear)
```

#### `query_key(button: int) -> None`

Request the current keymap for `button`. The badge responds with `EVT_KEY_REPLY`, which fires `on_key_reply`.

```python
badge.query_key(1)
# → fires badge.on_key_reply(1, mod, kc)
```

#### `set_effect_mode(mode: int) -> None`

Set the firmware LED effect mode (0=off, 1=rainbow-chase, 2=breathe). LED 4 is never affected by effects.

```python
from dc29.protocol import EffectMode

badge.set_effect_mode(EffectMode.BREATHE)
badge.set_effect_mode(EffectMode.OFF)
```

#### `set_button_flash(enabled: bool) -> None`

Enable or disable the white LED flash that occurs when a button is pressed. Default: enabled. RAM only.

```python
badge.set_button_flash(False)   # disable for bridge-controlled LEDs
badge.set_button_flash(True)    # re-enable
```

#### `send_raw(data: bytes) -> None`

Send arbitrary bytes to the badge. Use for custom protocol commands not covered by the methods above.

```python
badge.send_raw(bytes([0x01, ord('F'), 0]))   # disable button flash directly
```

#### `set_current_page(page: BridgePage | None) -> None`

Update `state.current_page` — the TUI reads this to show the active bridge name.

#### `close() -> None`

Stop the reader thread and close the serial port.

## `BadgeState` Dataclass

Broadcast via `on_state_change` whenever any observable state changes. The TUI subscribes to this single callback rather than wiring up all individual callbacks.

```python
@dataclass
class BadgeState:
    connected:    bool                        = False
    effect_mode:  int                         = 0
    mute_state:   MuteState                   = MuteState.NOT_IN_MEETING
    last_button:  int | None                  = None
    last_chord:   int | None                  = None
    key_map:      dict[int, tuple[int, int]]  = field(default_factory=dict)
    current_page: BridgePage | None           = None
```

## `BaseBridge` ABC

`dc29.bridges.base.BaseBridge` is the base class for all bridges. Source: `dc29/bridges/base.py`.

### Abstract Interface

```python
class BaseBridge(ABC):
    @property
    @abstractmethod
    def page(self) -> BridgePage: ...

    @abstractmethod
    async def run(self) -> None: ...

    async def handle_button(self, btn: int) -> None: ...
```

### Lifecycle Helpers

Call these from within `run()`:

```python
def _install_button_hook(self) -> None
    """Intercept button presses for buttons listed in self.page.buttons."""

def _uninstall_button_hook(self) -> None
    """Restore the previous button handler."""

def _apply_page_leds(self) -> None
    """Set all page button LEDs to their defined .led color."""

def _clear_page_leds(self) -> None
    """Turn off all page button LEDs."""
```

### Attributes

```python
self._badge: BadgeAPI
self._loop:  asyncio.AbstractEventLoop  # set by _install_button_hook
```

## `BridgePage` and `PageButton` Dataclasses

```python
@dataclass
class PageButton:
    label:        str                    # action name, e.g. "toggle-mute"
    led:          tuple[int, int, int]   # static idle color
    led_active:   tuple[int, int, int]   # color when action state is "on"
    led_inactive: tuple[int, int, int]   # color when action state is "off"

@dataclass
class BridgePage:
    name:        str                     # e.g. "teams"
    description: str
    buttons:     dict[int, PageButton]   # button number → behavior
```

`led_active` and `led_inactive` default to the same value as `led` if not specified.
